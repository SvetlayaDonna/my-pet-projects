1. Структура хранения данных

🗄️ ER-диаграмма

Визуальная схема базы данных, отображающая связи между таблицами.

Открыть ER-диаграмму на dbdiagram.io →

📋 Сокращения:

  • PK — primary key (первичный ключ)
  • FK — foreign key (внешний ключ)
  • unique — уникальное значение
  • not null — не может иметь нулевого значения
👤 users — Пользователи
Имя Тип Описание поля
id PK integer Идентификатор пользователя
username unique varchar Имя пользователя / Логин
phone varchar Телефон пользователя
🎫 bookings — Брони
Имя Тип Описание поля
id PK integer Идентификатор брони
user_id FK not null integer Идентификатор пользователя
sum decimal(12,2) Сумма брони по всем билетам
status enum Статус брони: 'active', 'canceled', 'expired', 'modified'
date_created timestamp Дата и время создания брони
date_modified timestamp Дата и время изменения брони
🏛️ halls — Залы
Имя Тип Описание поля
id PK integer Идентификатор зала
name varchar Название зала (например, «Зелёный зал»)
🎟️ tickets — Билеты
Имя Тип Описание поля
id PK integer Идентификатор билета
booking_id FK not null integer Идентификатор брони
session_id FK not null integer Идентификатор сеанса
seat_id FK not null integer Идентификатор места
status enum Статус билета: 'active', 'canceled', 'expired'
🔑 Индекс: unique_session_seat — уникальный индекс по полям (session_id, seat_id). Гарантирует, что одно и то же место нельзя забронировать дважды на один сеанс.
💺 seats — Места
Имя Тип Описание поля
id PK integer Идентификатор места
hall_id FK not null integer Идентификатор зала
row_number integer Номер ряда
seat_number integer Номер места в ряду
🔑 Индекс: unique_place — уникальный индекс по полям (hall_id, row_number, seat_number). Гарантирует, что в одном зале нет двух мест с одинаковыми координатами.
🎬 films — Фильмы
Имя Тип Описание поля
id PK integer Идентификатор фильма
name varchar Название фильма
genre varchar Жанр фильма
description text Описание фильма
age_rating enum Возрастное ограничение: '0+', '6+', '12+', '16+', '18+'
duration integer Продолжительность (в минутах)
🎥 sessions — Сеансы
Имя Тип Описание поля
id PK integer Идентификатор сеанса
hall_id FK not null integer Идентификатор зала
film_id FK not null integer Идентификатор фильма
start_datetime timestamp Дата и время начала сеанса
price decimal(12,2) Стоимость билета на сеанс

4. Ресурсы API и структура ответов

API обращается к следующим ресурсам:

  • sessions — сеансы
  • seats — места (подресурс: /sessions/{id}/seats)
  • bookings — брони

🎥 Ресурс: Сеансы (sessions)

Полная структура ответа для методов, возвращающих информацию о сеансе:

Имя Тип Описание поля
id integer Идентификатор сеанса
hall object Объект «Зал», состоит из:
  • id (integer) — идентификатор зала
  • name (string) — наименование зала
film object Объект «Фильм», состоит из:
  • id (integer) — идентификатор фильма
  • name (string) — название фильма
  • genre (string) — жанр
  • description (string) — описание
  • age_rating (enum: '0+', '6+', '12+', '16+', '18+') — возрастное ограничение
  • duration (integer) — продолжительность в минутах
start_datetime string (DateTime) Дата и время начала сеанса
seats array of objects Массив объектов «Места», каждый состоит из:
  • id (integer) — идентификатор места
  • row_number (integer) — номер ряда
  • seat_number (integer) — номер места
  • status (enum: 'free', 'booked') — статус места

🎫 Ресурс: Брони (bookings)

Полная структура ответа для методов, возвращающих информацию о брони:

Имя Тип Описание поля
id integer Идентификатор брони
user object Объект «Пользователь», состоит из:
  • id (integer) — идентификатор пользователя
  • username (string) — логин пользователя
  • phone (string, format: phone) — телефон пользователя
tickets array of objects Массив объектов «Билет», каждый состоит из:
  • id (integer) — идентификатор билета
  • session (object) — объект «Сеанс»:
    • id (integer)
    • hall (object: id, name)
    • film (object: id, name, genre, description, age_rating, duration)
  • seat (object) — объект «Место»:
    • id (integer)
    • row_number (integer)
    • seat_number (integer)
  • price (string, format: Money) — цена билета
  • status (enum: 'active', 'canceled', 'used') — статус билета
sum string (Money) Сумма брони по всем билетам
status enum Статус брони: 'active', 'canceled', 'expired', 'modified'
date_created string (DateTime) Дата и время создания брони
date_modified string (DateTime) Дата и время изменения брони

Отсутствуют

Поле Тип Описание

Отменить бронь

📥 Параметры запроса
Параметр Тип Описание
id (в пути) integer Идентификатор брони
📝 Тело запроса
Поле Тип Описание
status enum Новый статус брони: 'canceled'
📦 Параметры ответа
Параметр Тип Описание
id integer Идентификатор брони
username string Логин пользователя
tickets array Массив билетов в брони
sum string (Money) Сумма брони
status enum Новый статус брони: 'canceled'
date_created string (DateTime) Дата и время создания брони
date_modified string (DateTime) Дата и время отмены брони

🔧 Для администратора кинотеатра

PATCH /sessions/{id}

Изменить время сеанса

📥 Параметры запроса
Параметр Тип Описание
id (в пути) integer Идентификатор сеанса
📝 Тело запроса
Поле Тип Описание
start_datetime string (DateTime) Новая дата и время начала сеанса
📦 Параметры ответа
Параметр Тип Описание
id integer Идентификатор сеанса
username string Логин администратора, изменившего сеанс
start_datetime string (DateTime) Новая дата и время начала сеанса
💡 Примечание: Поле username добавлено в ответ, чтобы было видно, кто из трёх администраторов поменял время сеанса. Это обеспечивает прозрачность и аудит изменений.
