2. Структура таблиц базы данных

📋 Сокращения:

  • PK — primary key (первичный ключ)
  • FK — foreign key (внешний ключ)
  • unique — уникальное значение
  • not null — не может иметь нулевого значения

users (Пользователи)

Имя Тип Описание поля
id integer Идентификатор (PK)
username varchar Имя пользователя/Логин (unique)
phone varchar Телефон

bookings (Брони)

Имя Тип Описание поля
id integer Идентификатор (PK)
user_id integer Идентификатор пользователя (FK), not null
sum decimal(12,2) Сумма брони по всем билетам
status enum Статус брони: 'active', 'canceled', 'expired', 'modified'
date_created timestamp Дата и время создания брони
date_modified timestamp Дата и время изменения брони

halls (Залы)

Имя Тип Описание поля
id integer Идентификатор (PK)
name varchar Название зала

tickets (Билеты)

Имя Тип Описание поля
id integer Идентификатор (PK)
booking_id integer Идентификатор брони (FK), not null
session_id integer Идентификатор сеанса (FK), not null
seat_id integer Идентификатор места (FK), not null
status enum Статус билета: 'active', 'canceled', 'expired'
Индексы: unique_session_seat — уникальный индекс по полям (session_id, seat_id)

seats (Места)

Имя Тип Описание поля
id integer Идентификатор (PK)
hall_id integer Идентификатор зала (FK), not null
row_number integer Номер ряда
seat_number integer Номер места
Индексы: unique_place — уникальный индекс по полям (hall_id, row_number, seat_number)

films (Фильмы)

Имя Тип Описание поля
id integer Идентификатор (PK)
name varchar Название фильма
genre varchar Жанр фильма
description text Описание фильма
age_rating enum Возрастное ограничение: '0+', '6+', '12+', '16+', '18+'
duration integer Продолжительность (минуты)

sessions (Сеансы)

Имя Тип Описание поля
id integer Идентификатор (PK)
hall_id integer Идентификатор зала (FK), not null
film_id integer Идентификатор фильма (FK), not null
start_datetime timestamp Дата и время начала сеанса
price decimal(10,2) Цена билета

4. Эндпоинты API

Для пользователей

GET /sessions/{id}/seats
Узнать наличие мест на сеанс

Метод возвращает информацию о количестве мест на указанный сеанс. Только для авторизованных пользователей.

Параметры запроса:

Параметр Расположение Тип Обязательный Описание
id path integer Да Идентификатор сеанса
limit query integer Нет Количество мест на страницу (по умолчанию: 20)
offset query integer Нет Количество мест для пропуска (по умолчанию: 0)

Параметры ответа:

id, hall (object), film (object), start_datetime, seats (array with status)

POST /bookings
Забронировать место

Метод создаёт новую бронь на выбранные места в указанном сеансе. Только для авторизованных пользователей.

Тело запроса:

Поле Тип Описание
session.id integer Идентификатор сеанса
seats array Массив объектов мест (id, row_number, seat_number)

Параметры ответа:

id, user (object), tickets (array), sum, status, date_created

PATCH /bookings/{id}
Отменить бронь полностью

Метод полностью отменяет бронь. Только для авторизованных пользователей.

Параметры запроса:

Параметр Расположение Тип Обязательный
id path integer Да

Тело запроса:

Поле Тип Значение
status string "canceled" (Полная отмена брони)

Параметры ответа:

id, tickets (array), sum, status, date_created, date_modified

🔸 Для администраторов

PATCH /sessions/{id}
Изменить время сеанса

Метод изменяет время сеанса. Только для администраторов.

Параметры запроса:

Параметр Расположение Тип Обязательный
id path integer Да

Тело запроса:

Поле Тип Описание
start_datetime string (date-time) Новая дата и время начала сеанса

Параметры ответа:

id, user (username), start_datetime

Примечание: username добавлен для отслеживания, кто из администраторов изменил время сеанса.

5. Ресурсы API

API обращается к следующим ресурсам:

  • sessions — сеансы
  • seats — места (подресурс: /sessions/{id}/seats)
  • bookings — брони

6. Параметры запросов и ответов

📱 Для пользователя

5.1.1. Узнать наличие мест на сеанс

GET /sessions/{id}
Тип Параметры
Параметры запроса: id
Тело запроса: -
Параметры ответа: id, hall, film, start_datetime, array(seats) - массив мест (id, status)

5.1.2. Забронировать место

POST /bookings
Тип Параметры
Параметры запроса: -
Тело запроса: session(id), array(seat(id))
Параметры ответа: id, username, array(tickets), sum, status, date_created

5.1.3. Отменить бронь

PATCH /bookings/{id}
Тип Параметры
Параметры запроса: id
Тело запроса: status
Параметры ответа: id, username, array(tickets), sum, status, date_created, date_modified

🔧 Для администратора кинотеатра

5.2.1. Изменить время сеанса

PATCH /sessions/{id}
Тип Параметры
Параметры запроса: id
Тело запроса: start_datetime
Параметры ответа: id, username, start_datetime
Примечание: username добавлен, чтобы было видно, кто из трёх администраторов поменял время сеанса.
