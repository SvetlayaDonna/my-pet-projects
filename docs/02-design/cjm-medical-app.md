# Customer Journey Map (CJM): Мобильное приложение «Запись на приём»

**Проект:** Мобильное приложение для медицинских клиник  
**Цель:** Анализ пути пользователя от поиска клиники до управления записью  
**Методология:** CJM с выявлением болевых точек и точек роста

---

## 🎯 Этапы пользовательского пути

### Этап 1: Поиск информации и выбор клиники

| Параметр | Описание |
|---|---|
| **Действия клиента** | Ищет клинику в интернете |
| **Точки контакта** | Браузер (для первичного поиска клиники), сайт клиники |
| **Эмоции** | 😟 Неуверенность |
| **Проблемы** | Сложно сравнить клиники, нет отзывов |
| **Идеи улучшения** | 1. Выдавать в верхних строчках поисковых систем.<br>2. Сделать сайт клиники удобным для пользователя.<br>3. Показывать на сайте рейтинг клиники (интеграция с сайтом "ПроДокторов").<br>4. Добавить QR-код для скачивания мобильного приложения. |

---

### Этап 2: Скачивание приложения выбранной клиники

| Параметр | Описание |
|---|---|
| **Действия клиента** | Скачивает приложение, открывает раздел «Клиники», смотрит расположение на карте, выбирает удобную по расстоянию |
| **Точки контакта** | Google Play / RuStore, приложение (раздел «Клиники», карта) |
| **Эмоции** | 😐 Ожидание, лёгкое нетерпение |
| **Проблемы** | 1. Приложение занимает много места.<br>2. Непонятно, как добраться. |
| **Идеи улучшения** | 1. Сделать приложение максимально лёгким.<br>2. Показывать путь до клиники/клиник (если это сеть) - интеграция с Яндекс.Картами. |

---

### Этап 3: Заполнение профиля

| Параметр | Описание |
|---|---|
| **Действия клиента** | Заполняет свой профиль: вводит данные (ФИО, полис, дата рождения и т.д.) и, при необходимости, профиль членов семьи: ребёнок, пожилой родственник |
| **Точки контакта** | Приложение (свой профиль + профиль членов семьи (опционально)) |
| **Эмоции** | 😰 Нервозность (боится ошибиться в данных) |
| **Проблемы** | 1. Необходимо заполнять много полей.<br>2. Непонятно, как привязывать профили родственников к своему. |
| **Идеи улучшения** | 1. Пометить обязательные поля для заполнения.<br>2. Возле полей размещать подсказки по заполнению.<br>3. Сделать привязку дополнительных профилей интуитивно понятной. |

---

### Этап 4: Запись для себя или родственника

| Параметр | Описание |
|---|---|
| **Действия клиента** | Выбирает «Для себя» или «Для другого» (ребёнок, пожилой родитель) |
| **Точки контакта** | Приложение (форма записи) |
| **Эмоции** | 😕 Легкая неуверенность |
| **Проблемы** | Нет поля для особых потребностей (инвалидная коляска и т.д.) |
| **Идеи улучшения** | Добавить поле «Комментарий для клиники». |

---

### Этап 5: Выбор врача

| Параметр | Описание |
|---|---|
| **Действия клиента** | Ищет врача |
| **Точки контакта** | Приложение (карточка врача с отзывами) |
| **Эмоции** | 🤔 Заинтересованность, легкая неуверенность, напряжение |
| **Проблемы** | 1. Нет возможности выбора доктора по специальности/стажу/отзывам.<br>2. Нет возможности добавить врача в "Избранное".<br>3. Нет возможности проверить реальность отзывов. |
| **Идеи улучшения** | 1. Добавить фильтры по специальности/стажу/отзывам.<br>2. Добавить возможность отметить врача "звёздочкой", т.е. поместить в раздел "Избранное".<br>3. Показывать только проверенные отзывы: только отзывы пациентов, подтвердивших визит + интеграция с сайтом "ПроДокторов". |

---

### Этап 6: Выбор доступной даты и времени

| Параметр | Описание |
|---|---|
| **Действия клиента** | Смотрит расписание, выбирает свободную дату и время |
| **Точки контакта** | Приложение (календарь) |
| **Эмоции** | 🤔 Сомнение |
| **Проблемы** | 1. Нет возможности быстрого выбора даты.<br>2. Нет возможности выбрать тип приёма: первичный/вторичный. |
| **Идеи улучшения** | 1. На карточке врача сразу показывать ближайшую доступную дату.<br>2. Добавить возможность выбора разного типа приёмов: первичный/вторичный. |

---

### Этап 7: Выбор формата приема (онлайн / офлайн)

| Параметр | Описание |
|---|---|
| **Действия клиента** | Решает, нужен ли ему очный визит или онлайн-консультация |
| **Точки контакта** | Приложение (фильтр «Онлайн / Офлайн», карточка врача) |
| **Эмоции** | 🤔 Сомнение |
| **Проблемы** | Непонятно, какие услуги доступны онлайн |
| **Идеи улучшения** | Чек-лист «Когда подходит онлайн-прием» |

---

### Этап 8: Подтверждение записи

| Параметр | Описание |
|---|---|
| **Действия клиента** | Проверяет введенные данные, нажимает кнопку «Подтвердить запись», получает первое уведомление (в приложении) |
| **Точки контакта** | Приложение (экран подтверждения, in-app уведомление) |
| **Эмоции** | 😌 Облегчение, но легкая тревога (вдруг не записалось) |
| **Проблемы** | Нет возможности увидеть номер записи |
| **Идеи улучшения** | Показывать экран успеха с номером записи |

---

### Этап 9: Напоминания о записи

| Параметр | Описание |
|---|---|
| **Действия клиента** | Получает push / SMS / e-mail напоминание за 1 день и за 1 час до приема, может добавить событие в календарь телефона |
| **Точки контакта** | Push, SMS, e-mail, календарь ОС |
| **Эмоции** | 😊 Спокойствие, уверенность |
| **Проблемы** | Напоминания приходят не вовремя, нет ссылки на онлайн-прием в напоминании (если формат онлайн) |
| **Идеи улучшения** | Настраиваемое время напоминаний, ссылка на видеозвонок в push, кнопка «Добавить в календарь» |

---

### Этап 10: Управление записью (изменение / отмена)

| Параметр | Описание |
|---|---|
| **Действия клиента** | При необходимости отменяет или переносит запись через приложение, получает уведомление об отмене, возможно, видит альтернативные слоты |
| **Точки контакта** | Приложение (раздел «Мои записи», календарь) |
| **Эмоции** | 😤 Раздражение (если процесс сложный) или 😌 облегчение (если гибкий) |
| **Проблемы** | 1. Нет возможности быстро перенести запись без потери времени.<br>2. Не показываются альтернативные слоты при переносе.<br>3. Нет возможности переписать запись на другого пациента (маму вместо себя, например) |
| **Идеи улучшения** | 1. Кнопка «Перенести» с автоматическим подбором ближайших свободных слотов.<br>2. Возможность поменять пациента (если ошиблись при записи). |

---

## 📊 Анализ CJM

### Ключевые болевые точки:
1. **Этап 1** — Сложность сравнения клиник и отсутствие отзывов
2. **Этап 3** — Сложность заполнения профиля и привязки родственников
3. **Этап 5** — Отсутствие фильтрации и проверки отзывов
4. **Этап 10** — Сложность управления записью

### Точки роста:
- Интеграция с внешними сервисами (ПроДокторов, Яндекс.Карты)
- Упрощение UX (подсказки, обязательные поля, чек-листы)
- Гибкость управления записью (перенос, смена пациента)
- Персонализация (профили семьи, настраиваемые напоминания)

### Эмоциональная карта:
- **Негативные эмоции:** Этапы 1, 3, 5, 6, 7, 10
- **Нейтральные эмоции:** Этапы 2, 4
- **Позитивные эмоции:** Этапы 8, 9

3. **Приоритет №3:** Добавить гибкое управление записью (перенос, смена пациента)
