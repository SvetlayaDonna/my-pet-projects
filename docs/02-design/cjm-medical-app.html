Введение и методология

Что такое Customer Journey Map?

Customer Journey Map (CJM) — это визуализация пути клиента от первого контакта с продуктом до достижения его цели. Методология позволяет выявить болевые точки, эмоциональные спады и возможности для улучшения пользовательского опыта.

Цели анализа

  • Понять реальный опыт пользователя при записи к врачу
  • Выявить критические точки, где пользователь может отказаться от использования приложения
  • Найти возможности для улучшения UX и повышения конверсии
  • Сформировать приоритизированный список улучшений для разработки

10 Управление записью (изменение / отмена)
Действия клиента
При необходимости отменяет или переносит запись через приложение, получает уведомление об отмене, возможно, видит альтернативные слоты
Точки контакта
Приложение (раздел «Мои записи», календарь)
Эмоции
😤 Раздражение (если процесс сложный) или 😌 облегчение (если гибкий)
Проблемы
  • Нет возможности быстро перенести запись без потери времени
  • Не показываются альтернативные слоты при переносе
  • Нет возможности переписать запись на другого пациента (маму вместо себя, например)
Идеи улучшения
  • Кнопка «Перенести» с автоматическим подбором ближайших свободных слотов
  • Возможность поменять пациента (если ошиблись при записи)

4 Критических этапа

🎯 Ключевые болевые точки

Критично Этап 3 — Заполнение профиля: Нервозность пользователя из-за большого количества полей и непонятного процесса привязки родственников. Риск: отказ от завершения регистрации.
Критично Этап 5 — Выбор врача: Отсутствие фильтрации и проверки отзывов снижает доверие к приложению. Риск: пользователь уходит искать врача другими способами.
Важно Этап 10 — Управление записью: Сложность переноса/отмены записи вызывает раздражение. Риск: негативный отзыв в магазине приложений.
Важно Этап 1 — Поиск клиники: Сложность сравнения клиник на этапе до скачивания приложения. Риск: пользователь не доходит до установки.

💡 Приоритизированные рекомендации

Приоритет №1 Упростить процесс заполнения профиля и привязки родственников

Реализовать прогресс-бар, подсказки у полей, интуитивный интерфейс добавления членов семьи. Это снизит нервозность (этап 3) и повысит конверсию регистрации.

Приоритет №2 Внедрить интеграцию с ПроДокторов для доверия к отзывам

Показывать только проверенные отзывы пациентов, подтвердивших визит. Добавить фильтры по специальности, стажу, рейтингу. Это повысит уверенность пользователя при выборе врача (этап 5).

Приоритет №3 Добавить гибкое управление записью

Реализовать кнопку «Перенести» с автоматическим подбором слотов, возможность смены пациента в записи. Это снизит раздражение и повысит лояльность (этап 10).

Приоритет №4 Улучшить UX через подсказки и визуальную обратную связь

Добавить чек-листы, подсказки по формату ввода, экран успеха с номером записи. Это повысит уверенность пользователя на всех этапах.
