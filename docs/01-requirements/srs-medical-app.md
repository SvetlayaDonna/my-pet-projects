# SRS: Приложение «Запись на приём» для мед. клиники
**Стандарт:** ISO/IEC/IEEE 29148:2011  
**Версия:** 1.0 | **Дата:** 23.05.2026 | **Статус:** Черновик

## 1. Введение
### 1.1 Назначение
Документ описывает функциональные и нефункциональные требования к мобильному приложению «Запись на приём» для медицинских клиник.

### 1.2 Границы
* **Включает:** Требования к мобильному приложению (Android).
* **Не включает:** Архитектуру бэкенда и API (описаны в отдельных документах).

### 1.3 Ссылки и стандарты
| Наименование | Описание |
|---|---|
| ISO/IEC/IEEE 29148:2011 | Стандарт на написание требований к системам и ПО |
| Yandex MapKit SDK | Документация API карт |
| API ПроДокторов | Интеграция с сайтом отзывов |
| Нетрика (N3) | Интеграционная платформа для записи |

## 2. Общее описание
### 2.2 Функции системы
1. **Навигация:** Отображение списка клиник, маршрут (Яндекс.Карты), рейтинг (ПроДокторов).
2. **Профиль:** Заполнение профиля (ФИО, полис), привязка членов семьи.
3. **Выбор пациента:** «Для себя» или «Для другого».
4. **Поиск врача:** Фильтрация, «Избранное», отзывы (ПроДокторов).
5. **Запись:** Календарь слотов, формат (онлайн/офлайн), комментарий, напоминания.
6. **Управление записью:** Отмена, перенос, смена пациента.

### 2.4 Ограничения и зависимости
* **ОС:** Android 8.0 и выше, экран ≥ 4.5 дюймов.
* **Зависимости:** API Яндекс.Карт, API ПроДокторов, API Нетрика (N3), МИС клиники.

## 3. Функциональные требования (ФТ)
* **ФТ001-ФТ002:** Геолокация и построение маршрута до клиники.
* **ФТ003-ФТ004:** Заполнение профиля и управление профилями семьи.
* **ФТ005-ФТ006:** Выбор пациента для записи.
* **ФТ007-ФТ010:** Поиск врача, карточка врача, отзывы, «Избранное».
* **ФТ011-ФТ017:** Календарь слотов, выбор формата, чек-лист онлайн, подтверждение записи.
* **ФТ018-ФТ020:** Push/SMS/e-mail напоминания, ссылка на видеозвонок, добавление в календарь ОС.
* **ФТ021-ФТ024:** Отмена/перенос записи, история приёмов.

## 4. Нефункциональные требования (НФТ)
* **Производительность:** Загрузка экранов ≤ 5 сек, маршрут ≤ 2 сек. Память ≤ 150 МБ.
* **Надёжность:** Uptime 99.9%. Работа с кэшем при обрыве сети.
* **UX:** Обязательные поля помечены `*`. Адаптивный дизайн 4.5–7 дюймов. Блокировка недоступных слотов.

## 5. Внешние интерфейсы
| Интерфейс | Назначение | Формат | Протокол |
|---|---|---|---|
| API «Нетрика (N3)» | Запись, расписание, слоты | JSON | HTTPS |
| API «ПроДокторов» | Рейтинги и отзывы | JSON | HTTPS |
| Yandex MapKit SDK | Карты, маршруты, геокодинг | - | HTTPS |
| Android Calendar Provider | Добавление событий в календарь | - | - |

## 6. Критерии приёмки
1. Успешное прохождение всех функциональных тестов.
2. Время отображения информации ≤ 5 секунд.
3. Корректная работа на Android 8.0+ и экранах ≥ 4.5".
4. Определение геолокации при каждом запуске.
