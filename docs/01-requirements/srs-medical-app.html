📋 SRS: Мобильное приложение «Запись на приём»

Стандарт: ISO/IEC/IEEE 29148:2011

Версия: 1.0

Дата: 23.05.2026

Статус: Черновик

Идентификатор: MAA-SRS-01

1. Введение

1.1 Назначение документа

Настоящий документ «Спецификация программных требований» (SRS) полностью описывает функциональные и нефункциональные требования к мобильному приложению «Запись на приём» для медицинских клиник.

1.2 Границы и область применения

В данный SRS входят:

  • Требования к мобильному приложению «Запись на приём» для Android

Вне границ данного SRS остаются:

  • Реализация и архитектура серверной части (бэкенд)
  • API для интеграции с МИС и внешними сервисами

2. Функциональные требования

2.1 Навигация и информация о клинике

  • ФТ001: Определение геолокации пользователя
  • ФТ002: Отображение маршрута до клиники (интеграция с Яндекс.Картами)

2.2 Управление профилем

  • ФТ003: Заполнение профиля (ФИО, полис, дата рождения)
  • ФТ004: Добавление профилей членов семьи

2.3 Поиск и выбор врача

  • ФТ007: Фильтрация по специальности/стажу/отзывам
  • ФТ008: Карточка врача с отзывами (интеграция с ПроДокторов)
  • ФТ009: Добавление в «Избранное»

2.4 Запись к врачу

  • ФТ011: Календарь с доступными слотами
  • ФТ013: Выбор формата приёма (онлайн/офлайн)
  • ФТ015: Поле «Комментарий для клиники»
  • ФТ018: Push/SMS/e-mail напоминания

3. Нефункциональные требования

3.1 Производительность

  • НФТ001: Отображение экранов ≤ 5 секунд
  • НФТ002: Построение маршрута ≤ 2 секунды
  • НФТ003: Потребление памяти ≤ 150 МБ

3.2 Надёжность

  • НФТ005: Uptime ≥ 99.9%
  • НФТ004: Работа при обрыве соединения (кэширование)

4. Внешние интерфейсы

Интерфейс Назначение Формат Протокол
API Нетрика (N3) Запись на приём, расписание JSON HTTPS
API ПроДокторов Рейтинги и отзывы JSON HTTPS
Yandex MapKit SDK Карты, маршруты - HTTPS
