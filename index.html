🛒 Sequence Diagram: Поиск и заказ товара в интернет-магазине

Моделирование процесса взаимодействия компонентов системы при поиске и заказе товара.

Что сделано:

  • ✅ Построена диаграмма последовательности (Sequence Diagram)
  • ✅ Описаны 3 основных процесса: Авторизация, Поиск товара, Заказ товара
  • ✅ Показано взаимодействие компонентов: User Interface, Authorization Server, Server, DataBase, Система логирования
  • ✅ Учтены альтернативные сценарии (успешная/неуспешная авторизация)
  • ✅ Описаны параллельные процессы (par) и циклы (loop)

Основные процессы:

  1. Авторизация: ввод логина/пароля, проверка учётных данных, логирование
  2. Поиск товара: GET-запрос, select-запрос в БД, возврат JSON
  3. Заказ товара: POST-запрос, insert в БД, обновление статуса (каждые 5 минут)

Компоненты системы:

  • Internet (User Interface)
  • Authorization Server (Authorization-API)
  • Server (Catalog-API, Order-API)
  • DataBase (Catalog DB, Order DB)
  • Система логирования
