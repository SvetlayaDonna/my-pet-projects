### 🏥 1. Мобильное приложение «Запись на приём» (MedTech)

Комплексный проект по созданию Android-приложения для записи в медицинские клиники.

**Что сделано:**
- ✅ Написана **спецификация требований (SRS)** по международному стандарту **ISO/IEC/IEEE 29148:2011**
- ✅ Сформирован **бэклог проекта** (29 задач) с приоритизацией и план-графиком по 4 спринтам
- ✅ Построена **Customer Journey Map (CJM)** с анализом 10 этапов пользовательского пути
- ✅ Выявлены **ключевые болевые точки** и предложены **решения для улучшения UX**
- ✅ Проведён **анализ требований к внешним интеграциям**: Яндекс.Карты, ПроДокторов, Нетрика N3, МИС клиники
- ✅ Описаны **24 функциональных требования** и **14 нефункциональных требований**
- ✅ Учтены требования к производительности, надёжности (Uptime 99.9%), UX/UI

**Ключевой функционал:**
- Запись к врачу онлайн/офлайн с выбором слотов
- Геолокация и построение маршрута до клиники
- Профили членов семьи для записи
- Push/SMS/e-mail напоминания
- Интеграция с системным календарём Android
- Фильтрация врачей по специальности, стажу, отзывам
- Управление записью (отмена, перенос, смена пациента)

**Анализ пользовательского опыта (CJM):**
- Проанализировано **10 этапов** пути клиента
- Выявлено **15+ болевых точек**
- Предложено **20+ улучшений** UX
- Построена **эмоциональная карта** пользователя

### 🖨️ 2. Терминал самообслуживания PhotoKiosk (IoT)

Разработка требований для ПО киоска печати фотографий из Instagram и с телефона.

**Что сделано:**
- ✅ Проведён **анализ требований** с формализацией (БТ, ПТ, ФТ, БП, СТ, АК, ВСиИ, Огр)
- ✅ Детально описаны **сценарии обработки системных ошибок** (6 категорий ошибок)
- ✅ Прописаны **системные требования** к аппаратному и программному обеспечению
- ✅ Определены **атрибуты качества**: время отклика ≤ 200 мс, загрузка превью ≤ 2 сек

**Ключевой функционал:**
- Печать фото из Instagram (через API) и с телефона (USB)
- Выбор количества копий (1-100)
- Оплата через NFC-терминал
- Автоматическая безопасность сессий (авто-logout из IG через 5 мин)

**Технологии:** Linux/Windows IoT, Instagram API, ESC/POS, OPOS/JavaPOS

### 🎬 3. REST API для кинотеатра «Искорка»

Проектирование бэкенда для системы бронирования билетов в кинотеатр.

**Что сделано:**
- ✅ Спроектирована **ER-модель базы данных** (7 таблиц: users, bookings, halls, tickets, seats, films, sessions)
- ✅ Разработана **OpenAPI 3.0 спецификация** (Swagger) с полным описанием эндпоинтов
- ✅ Описаны **схемы данных** (Sessions, Bookings, Seats, Films, Halls, Users)
- ✅ Определена **ролевая модель**: пользователи и администраторы
- ✅ Прописаны **коды ответов** (200, 400, 401, 403, 404, 500) с примерами

**Эндпоинты API:**
- `GET /sessions/{id}/seats` — наличие мест на сеанс
- `POST /bookings` — забронировать место
- `PATCH /bookings/{id}` — отменить бронь
- `PATCH /sessions/{id}` — изменить время сеанса (админ)

**Технологии:** OpenAPI 3.0, REST, JSON, HTTPS

📄 [Проектирование API и БД](docs/02-design/api-design-cinema.md) | [OpenAPI спецификация](docs/03-api/openapi-cinema.yaml)

### 🛒 5. Sequence Diagram: Поиск и заказ товара в интернет-магазине

Моделирование процесса взаимодействия компонентов системы при поиске и заказе товара.

**Что сделано:**
- ✅ Построена **диаграмма последовательности** (Sequence Diagram)
- ✅ Описаны **3 основных процесса**: Авторизация, Поиск товара, Заказ товара
- ✅ Показано **взаимодействие компонентов**: User Interface, Authorization Server, Server, DataBase, Система логирования
- ✅ Учтены **альтернативные сценарии** (успешная/неуспешная авторизация)
- ✅ Описаны **параллельные процессы** (par) и **циклы** (loop)

**Основные процессы:**
1. **Авторизация**: ввод логина/пароля, проверка учётных данных, логирование
2. **Поиск товара**: GET-запрос, select-запрос в БД, возврат JSON
3. **Заказ товара**: POST-запрос, insert в БД, обновление статуса (каждые 5 минут)
